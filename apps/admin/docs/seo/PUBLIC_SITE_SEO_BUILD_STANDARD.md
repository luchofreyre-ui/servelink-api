# Public-site SEO build standard

Permanent build rule for every new public-facing page or feature.

## For every new public-facing page/feature

1. **Identify SEO opportunity**  
   - Target query intent (informational, commercial, local).  
   - Primary and secondary keywords.  
   - Entity (service, city, neighborhood, topic).

2. **Define URL target**  
   - Follow the URL architecture (service → city → neighborhood).  
   - One canonical URL per logical page.  
   - No duplicate content under different URLs unless properly canonicalized or noindexed.

3. **Define title / meta / H1**  
   - **Title:** Unique, descriptive, within length limits; include primary keyword and brand where appropriate.  
   - **Meta description:** Unique, within length limits; include CTA or clarity on page purpose.  
   - **H1:** One per page; matches or closely aligns with title and target query.

4. **Define schema opportunity**  
   - Choose applicable types (LocalBusiness, FAQPage, BreadcrumbList, Review, etc.).  
   - Implement only with accurate, non-misleading data.

5. **Define internal links**  
   - Parent/child and sibling links per the internal linking model.  
   - Anchor text that is descriptive and not over-optimized.

6. **Define conversion CTA**  
   - Clear next step (book, get quote, contact) above the fold and repeated where appropriate.  
   - CTA should be consistent with the page’s place in the funnel.

7. **Define performance constraints**  
   - Core Web Vitals and LCP/CLS targets.  
   - Minimal blocking resources; images and critical assets optimized.

## Checklist before launch

- [ ] URL matches architecture and is canonical.  
- [ ] Title, meta description, and H1 are set and unique.  
- [ ] Schema is implemented only where data is accurate.  
- [ ] Internal links and CTAs are in place.  
- [ ] Performance constraints are met or documented for follow-up.
